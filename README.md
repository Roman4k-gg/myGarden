# MyGarden Backend

Микросервисный бэкенд для приложения MyGarden, спроектированный с упором на масштабируемость и производительность. Архитектура использует событийно-ориентированный подход и gRPC для быстрого внутреннего обмена данными.

## Архитектура

Система состоит из нескольких независимых микросервисов, взаимодействующих через gRPC и брокеры сообщений:

- **API Gateway**: RESTful точка входа для всех клиентов. Обрабатывает маршрутизацию, валидацию запросов и JWT авторизацию.
- **User Service**: Управляет регистрацией пользователей, аутентификацией (генерация JWT) и профилями пользователей.
- **Catalog Service**: Управляет каталогом растений и избранным пользователей. Реализует паттерн Cache-Aside через Redis для быстрой выдачи данных.
- **Notification Service**: Фоновый воркер, который асинхронно обрабатывает события (например, добавление в избранное) из топиков Kafka.

## Стек технологий

- **Язык**: Go (Golang) 1.23+
- **База данных**: PostgreSQL (pgx)
- **Кэш**: Redis
- **Брокер сообщений**: Apache Kafka & Zookeeper
- **RPC Фреймворк**: gRPC / Protocol Buffers
- **HTTP Фреймворк**: Gin Gonic
- **Инфраструктура**: Docker & Docker Compose

## Быстрый старт

Вся инфраструктура, включая базы данных, брокеры сообщений и сами микросервисы, полностью докеризована.

```bash
docker-compose up -d --build
```

Дождитесь инициализации базы данных, применения миграций и запуска Kafka. API Gateway будет доступен по адресу `http://localhost:3000`.

## API Эндпоинты

### Аутентификация
- `POST /api/v1/register` - Регистрация нового пользователя
- `POST /api/v1/login` - Аутентификация и получение JWT токена

### Каталог и Избранное (Защищено: Требуется `Authorization: Bearer <token>`)
- `GET /api/v1/plants` - Получение каталога всех растений
- `POST /api/v1/favorites?plant_id=<id>` - Добавление растения в избранное
- `GET /api/v1/favorites` - Получение всех избранных растений для авторизованного пользователя

## Разработка

Для перегенерации gRPC стабов после изменения `.proto` файлов используйте следующие команды:

```bash
protoc --go_out=pkg/user_v1 --go-grpc_out=pkg/user_v1 proto/user/user.proto
protoc --go_out=pkg/catalog_v1 --go-grpc_out=pkg/catalog_v1 proto/catalog/catalog.proto
```
